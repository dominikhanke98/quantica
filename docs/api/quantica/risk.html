quantica.risk

Market-risk engines and the backtesting/validation layer (Phase 3).

The design intent (CLAUDE.md §1, effective-challenge identity) is that the backtesting layer — not the VaR/ES numbers — is the deliverable: a risk model is only as good as the evidence that it is adequately calibrated, and this package ships both the standard VaR backtests and the harder-to-get Expected-Shortfall backtest (Acerbi--Székely), with the tests validating the backtests themselves (size and power) in tests/risk.

class FilteredHistoricalSimulationVaR:

FHS VaR/ES: a GARCH(1,1) volatility filter over bootstrapped residuals.

Fits a constant-mean GARCH(1,1) to the portfolio return series, standardises by the conditional volatility, and rebuilds a one-step-ahead loss sample as \( \hat\mu + \hat\sigma_{T+1}\, z_i \) over the standardised residuals \( z_i \). The empirical quantile of that sample is the FHS VaR/ES — a nonparametric tail riding on a dynamic volatility (Barone-Adesi et al., 1999).

Parameters
  • p, q (int, optional): GARCH orders (default 1, 1).
Notes

Requires the optional arch dependency; the import is deferred so the rest of quantica.risk works without it.

class ParametricVaR:

Variance--covariance VaR/ES under a multivariate-normal return assumption.

Uses the sample mean vector \( \mu \) and covariance matrix \( \Sigma \) to form the portfolio P&amp;L moments \( \mu_p = V\,w^\top\mu \), \( \sigma_p = V\sqrt{w^\top \Sigma w} \), then the Gaussian closed form. The normality assumption is documented and is exactly what the backtests challenge.

def acerbi_szekely( realized_losses: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], var_forecasts: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], es_forecasts: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], level: float, *, method: str = 'Z2', null_losses: numpy.ndarray[tuple[Any, ...], numpy.dtype[numpy.float64]] | None = None) -> AcerbiSzekelyResult:

Acerbi--Székely (2014) Expected-Shortfall backtest.

Parameters
  • realized_losses (ndarray): Realised losses \( L_t \), shape (T,).
  • var_forecasts, es_forecasts (ndarray): The VaR and ES forecasts aligned with realized_losses (scalars are broadcast).
  • level (float): Confidence level \( \alpha \).
  • method ({"Z1", "Z2"}): "Z2" (unconditional, the default workhorse) averages the tail loss over all days; "Z1" (conditional) averages only over exception days.
  • null_losses (ndarray, optional): Shape (n_sims, T) of losses simulated under the predictive model (i.e. under \( H_0 \)). When given, the p-value is the fraction of null statistics at least as large as the realised one (ES not elicitable → the null is obtained by simulation, not a closed form).
def basel_traffic_light( n_exceptions: int, n_obs: int = 250, level: float = 0.99) -> BaselResult:

Basel green/yellow/red zone and capital multiplier add-on.

Zones follow the cumulative probability of observing up to n_exceptions exceptions under the null rate \( 1-\alpha \): green below 95%, yellow from 95% up to 99.99%, red at or above 99.99%. The multiplier add-on is the supervisory plus-factor (0 in green; the standard 0.40-0.85 table in yellow; 1.00 in red), calibrated for the canonical 250-day / 99% window.

def book_pla_test( book: OptionBook, scenarios: MarketScenarios, *, rtpl_method: Literal['delta-normal', 'delta-gamma'] = 'delta-gamma') -> PLAResult:

Run PLA on an option book: HPL = full revaluation, RTPL = sensitivities P&L.

Makes the pricing/risk connection concrete — the HPL is the pricing path (OptionBook.full_revaluation_pnl(), no drift from the pricers) and the RTPL is the risk model's sensitivities P&L on the same seeded scenarios. A "delta-gamma" risk model spans an option book's curvature; a "delta-normal" (delta-only) model does not, and short-gamma desks fail.

def book_var_es( book: OptionBook, scenarios: MarketScenarios, level: float, *, method: Literal['full', 'delta-normal', 'delta-gamma'] = 'full') -> RiskEstimate:

VaR/ES of an option book under scenarios, by the chosen revaluation method.

A thin adapter into the existing risk layer: compute the book's scenario P&amp;L, negate it into losses, and hand it to ~quantica.risk.measures.empirical_var_es() — the risk/backtest machinery is untouched (the P&amp;L-series seam doing its job). Evaluating all three methods on the same scenario set isolates approximation error from sampling noise.

def christoffersen_cc( hits: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], level: float) -> ChristoffersenResult:

Christoffersen conditional-coverage test: coverage and independence.

The statistic is Kupiec's LR plus the independence LR and is \( \sim \chi^2_2 \) (Christoffersen, 1998).

def christoffersen_independence( hits: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]]) -> ChristoffersenResult:

Christoffersen test that exceptions are serially independent (\( \chi^2_1 \)).

A first-order Markov alternative (different exception probabilities after a hit vs a non-hit) is tested against the i.i.d. null; a large statistic means exceptions cluster.

def empirical_var_es( losses: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], level: float, *, method: str = 'historical') -> RiskEstimate:

Empirical VaR and ES from a sample of losses.

VaR is the linear-interpolated \( \alpha \)-quantile of losses. ES uses the Rockafellar--Uryasev tail-mean identity

$$\mathrm{ES}_\alpha = \mathrm{VaR}_\alpha + \frac{1}{1-\alpha}\,\mathbb E\big[(L - \mathrm{VaR}_\alpha)^+\big],$$

which is numerically stable (no reliance on how many points land exactly in the tail) and coincides with the average of the worst \( (1-\alpha) \) fraction of losses for a continuous distribution.

Parameters
  • losses (ndarray): Sample of losses (positive = loss).
  • level (float): Confidence level \( \alpha \).

def normal_var_es( mean: float, sigma: float, level: float, *, method: str = 'parametric-normal') -> RiskEstimate:

Gaussian closed-form VaR and ES for P&amp;L \( \sim \mathcal N(\mu, \sigma^2) \).

With losses \( L = -\text{P&amp;L} \sim \mathcal N(-\mu, \sigma^2) \) and \( z_\alpha = \Phi^{-1}(\alpha) \),

$$\mathrm{VaR}_\alpha = -\mu + \sigma z_\alpha, \qquad \mathrm{ES}_\alpha = -\mu + \sigma\,\frac{\phi(z_\alpha)}{1 - \alpha}.$$

Parameters
  • mean (float): Mean of the P&amp;L (drift), in money units.
  • sigma (float): Standard deviation of the P&amp;L. Must be non-negative.
  • level (float): Confidence level \( \alpha \).
def pla_test( rtpl: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], hpl: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]]) -> PLAResult:

Run the FRTB P&L-attribution test on aligned RTPL and HPL series.

The desk's overall zone is the worse of the two per-metric zones: green only if both metrics are green, red if either metric is red, amber otherwise (BCBS d457, MAR33.43--33.46).

For each day t >= window the engine is refit on the trailing window days [t-window, t) and its VaR/ES forecast is compared with the realised loss on day t.

Returns

def spearman_correlation( rtpl: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]], hpl: numpy.ndarray[tuple[typing.Any, ...], numpy.dtype[numpy.float64]]) -> float:

Spearman rank correlation of paired (RTPL, HPL) observations.

Computed as the Pearson correlation of the (tie-averaged) ranks — the definition of Spearman's rho. Matches scipy.stats.spearmanr() (asserted in the tests). If either series is constant (no rank information — e.g. a perfectly delta-hedged book scored by a delta-only risk model), the correlation is undefined and reported as 0.0, which lands in the red zone.
